# SKILL: design_identity_master

## OBJETIVO
Definir y aplicar una identidad visual única, consistente y profesional en toda la aplicación.

Esta skill convierte la app en un producto SaaS vendible, no en un sistema genérico.

---

## PRINCIPIO BASE

Cada pantalla, componente y elemento debe parecer parte del mismo sistema.

PROHIBIDO:
- estilos inconsistentes
- colores aleatorios
- tipografías mezcladas
- UI genérica sin identidad

---

## IDENTIDAD VISUAL BASE

### 🎨 PALETA DE COLORES (OBLIGATORIA)

- Primary: #3B82F6 (azul tecnológico)
- Secondary: #1E293B (superficie)
- Background: #0F172A (fondo oscuro)
- Accent: #22C55E (éxito / puntualidad)
- Warning: #F59E0B (alertas)
- Danger: #EF4444 (errores / ausencias)

### Reglas:

- Usar máximo 2 colores principales por pantalla
- Mantener contraste alto
- Evitar colores saturados innecesarios

---

## 🔤 TIPOGRAFÍA

- Principal: Inter o Poppins

### Jerarquía obligatoria:

- H1 → 28–32px / bold
- H2 → 22–24px / semibold
- H3 → 18–20px / medium
- Body → 14–16px
- Labels → 12–13px

### Reglas:

- NO mezclar fuentes
- Mantener consistencia en tamaños
- Usar peso tipográfico para jerarquía (no solo color)

---

## 🧩 COMPONENTES BASE (SISTEMA)

Todos los componentes deben seguir este estilo:

### Cards
- Fondo: Secondary (#1E293B)
- Bordes redondeados (12–16px)
- Sombra suave
- Padding consistente

### Botones
- Primary: azul sólido
- Secondary: outline
- Hover: cambio sutil (no brusco)

### Inputs
- Fondo oscuro
- Border suave
- Focus con glow azul

### Tablas
- Limpias
- Espaciado cómodo
- Hover en filas

---

## 🌌 ESTILO VISUAL

Inspiración:

- SaaS moderno
- dashboards empresariales premium
- estilo tecnológico (no corporativo aburrido)

### Reglas:

- UI limpia (espacio negativo)
- No saturar con información
- Usar iconografía simple

---

## ✨ MICROINTERACCIONES

- Hover en botones
- Transiciones suaves (150–250ms)
- Feedback inmediato en acciones

PROHIBIDO:
- animaciones exageradas
- efectos distractores

---

## 📐 ESPACIADO Y LAYOUT

Sistema base: 8px grid

- Márgenes consistentes
- Separación clara entre bloques
- Layout estructurado

---

## 🧠 CONSISTENCIA GLOBAL

Antes de generar cualquier UI:

1. Verificar que respeta la paleta
2. Verificar tipografía
3. Verificar componentes
4. Verificar jerarquía visual

Si no cumple → REGENERAR

---

## 🔥 DIFERENCIACIÓN (IMPORTANTE)

La UI debe transmitir:

- tecnología avanzada
- confiabilidad
- control empresarial

NO debe parecer:
- plantilla gratuita
- dashboard genérico
- sistema viejo

---
